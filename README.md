专治外网大文件下载慢、易中断、有反爬或防盗链限制等“下载困难”痛点。基于 DrissionPage 底层接管浏览器与 curl_cffi 指纹伪装，采用 16 线程分块并发下载，原生支持断点续传。半自动化硬核下载工具，无视常规网络拦截。
# 📦 半自动多线程下载工具（需手动抓取下载请求）

## 📌 项目说明

这是一个**半自动下载工具**，核心作用是：

👉 帮你把“浏览器里的真实下载链接”抓出来
👉 然后用多线程方式把文件高速下载下来

⚠️ 注意：
这个工具**不是一键下载器**，需要你**手动配合抓包 / 点击下载按钮**。

---

## 🧠 工具到底干嘛的（说人话版）

正常你在浏览器下载文件：

* 点下载 → 浏览器慢慢下
* 有时候还限速 / 不稳定

这个工具做的是：

1. 你自己打开下载页面
2. 手动点击下载按钮
3. 程序帮你**抓到真实下载链接（302跳转后的）**
4. 然后用 **16线程并发** 帮你把文件“拆开下载”
5. 最后自动拼回完整文件

👉 本质就是：**接管浏览器下载 → 自己加速下载**

---

## ⚙️ 使用前你必须知道的

这个工具需要你做三件事：

### 1️⃣ 自己填下载页面地址

```python
page.get("这里填你要下载的网页")
```

👉 比如：

```python
page.get("https://desktop.github.com/")
```

---

### 2️⃣ 自己手动点击下载按钮

程序不会帮你点按钮，你必须：

👉 打开页面 → 自己点 “Download”

程序才会抓到请求。

---

### 3️⃣ 不会抓包？用 F12 或直接问 AI

代码里已经写了提示：

👉 F12 → Network → 找 302 请求
👉 或者直接问 AI：“这个网站下载链接怎么抓”

---

## 🛠️ 使用步骤（完整流程）

### 第一步：安装依赖

```bash
pip install curl_cffi DrissionPage
```

---

### 第二步：改路径（必须改）

```python
SAVE_PATH = "你要保存到哪里"
TEMP_DIR = "临时文件放哪里"
```

比如：

```python
SAVE_PATH = r"D:\Download\Final"
TEMP_DIR = r"D:\Download\Temp"
```

---

### 第三步：填下载页面

```python
page.get("填网址")
```

---

### 第四步：运行

```bash
python 文件名.py
```

---

### 第五步：手动操作（关键）

程序启动后：

1. 浏览器会打开
2. 你自己点击下载按钮
3. 程序开始抓请求
4. 自动开始高速下载

---

## ⚡ 参数说明

```python
MAX_THREADS = 16
```

* 越大 → 越快
* 但太大可能被限速 / 失败

---

## ⚠️ 注意事项（重要）

* ❗ 必须手动点击下载
* ❗ 必须自己填网址
* ❗ 不是全自动工具
* ❗ 服务器必须支持 Range 下载
  

---

## 📁 下载结果

最终文件：

👉 在 `SAVE_PATH` 目录

临时碎片：

👉 在 `TEMP_DIR`（下载完会删除）
