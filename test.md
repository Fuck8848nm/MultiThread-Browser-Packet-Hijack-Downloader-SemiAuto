# 📦 多线程下载工具（基于浏览器请求捕获）

## 📌 项目介绍

本项目是一个基于 Python 编写的**多线程下载工具**，结合浏览器请求监听与 HTTP 分块下载技术，实现对大文件的稳定下载与加速处理。

项目核心能力：

* 通过浏览器监听获取真实下载链接
* 自动处理重定向后的最终下载地址
* 使用 HTTP Range 进行分块下载
* 多线程并发提升下载速度
* 自动合并下载文件

适用于以下场景：

* 大文件下载速度优化
* 下载过程不稳定时的恢复处理
* 网络请求与下载机制学习研究

---

## ⚙️ 功能特点

* ✅ 多线程并发下载（默认 16 线程）
* ✅ 支持断点续传（基于分块文件）
* ✅ 自动合并分片文件
* ✅ 浏览器请求监听（自动获取下载链接）
* ✅ 自定义请求头（模拟浏览器环境）

---

## 🧠 工作流程

程序运行流程如下：

1. 启动本地 Chromium 内核浏览器（Edge）
2. 监听页面网络请求
3. 用户手动点击下载按钮
4. 程序捕获下载请求 URL
5. 获取文件大小（Content-Length）
6. 按线程数拆分下载区间
7. 多线程并发下载
8. 下载完成后自动合并文件

## 🛠️ 使用方法

### 1️⃣ 安装依赖

```bash
pip install curl_cffi DrissionPage
```

---

### 2️⃣ 修改配置

在代码顶部设置路径：

```python
SAVE_PATH = "你的最终保存路径"
TEMP_DIR = "临时文件路径"
MAX_THREADS = 16
```

示例：

```python
SAVE_PATH = r"D:\Download\Final"
TEMP_DIR = r"D:\Download\Temp"
```

---

### 3️⃣ 运行程序

```bash
python 文件名.py
```

运行后：

* 程序会自动打开浏览器
* 进入你设置的目标页面
* **手动点击下载按钮**
* 程序自动捕获下载请求并开始下载

---

## ⚠️ 使用注意事项

* 必须手动触发下载（程序不会自动点击）
* 目标服务器必须支持 `HTTP Range`
* 线程数过高可能导致连接失败或限速
* 建议根据网络环境合理调整线程数

---

## 🔧 可修改内容

你可以根据需求修改：

* 线程数量（MAX_THREADS）
* 保存路径（SAVE_PATH）
* 临时目录（TEMP_DIR）
* 请求头（User-Agent / Referer）

---

## ❌ 不建议修改

以下部分不建议随意改动：

* Range 分块逻辑
* 分片合并顺序
* 下载重试机制

否则可能导致文件损坏或下载失败。

---

## 📁 输出说明

* 最终文件保存在：`SAVE_PATH`
* 临时文件保存在：`TEMP_DIR`
* 下载完成后临时文件会自动清理
